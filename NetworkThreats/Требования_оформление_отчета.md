## Введение

В данном документе приведены примеры структуры и правил оформления отчета по курсовой по всем основным разделам. Также данный документ является эталонным примером оформления (если оформление вашей работы отличается от этого документа, то на проверке это будет указано как ошибка и снизит балл или будет основанием не принять работу).

**Основные требования, которые предъявляются к работе:**

- Титульный лист должен быть оформлен по правилам университета.
- Основной текст должен содержать:
  - формальное описание созданных классов и их взаимосвязь, примеры работы программы (скриншоты работы программы);
  - основной текст не включает в себя: содержание, список источников и приложения.
- Обязательное наличие затекстовых ссылок на источники в тексте.
- Список источников не менее 3.

## 1. Оформление заголовков, разделов и подразделов

Все заголовки, разделы и подразделы в отчете оформляются одинаково и не допускается «зоопарка стилей».

**Правила оформления заголовков:**

- Шрифт Times New Roman
- Размер 14
- Межстрочный интервал 1,5
- Выровнен по левому краю
- Полужирный
- Каждый раздел начинается с нового листа, чтобы начать новый раздел **допускается пропуск не более половины листа** предыдущего раздела
- **Раздел не должен состоять из 1 листа!** (Исключение — этот раздел, так как он даётся для примера оформления)
- Между текстом подразделов пропускается 1 строка
- После заголовка раздела и подраздела никаких пропусков строк
- Не допускается **«одиноких заголовков»** — заголовок не должен быть без текста (пример: заголовок является последней строкой листа)

Раздел является законченной смысловой единицей, в которой повествуется о какой-либо теме. Если возникает необходимость разбить текст на смысловые или логические блоки в пределах одной темы — необходимо использовать подразделы. В одном разделе может быть любое количество подразделов любой вложенности.

Все разделы нумеруются, нумерация начинается с 1. Подразделы **также нумеруются**, начиная с 1, в качестве префикса выступает номер раздела.

## 3. Оформление основных объектов

### 3.1 Оформление текста

Оформление основного текста отчета включает в себя следующие правила:

- Все абзацы начинаются с красной строки 1,25 см
- Не допускается между абзацами пустых промежутков и пропусков строк
- Шрифт Times New Roman
- Размер 14
- Межстрочный интервал 1,5
- Выровнен по ширине
- Поля документа:
  - 2 см снизу
  - 2 см сверху
  - 3 см слева
  - 1,5 см справа

### 3.2 Оформление медиа объектов

В отчете может потребоваться вставка медиа объектов: рисунки, таблицы, программный код и т.д. Подобные объекты должны вставляться в текст только когда они:

- Поясняют какую-либо информацию лучше или проще чем текст (пример: блок-схема алгоритма, график)
- Соответствуют теме текста

**Общие правила оформления для всех медиа объектов:**

- Объекты не должны добавляться «красивые», «просто так», «для объёма» (пример: логотип компании, библиотеки, программы)
- Не должно существовать «непонятных» объектов, которые не относятся к тексту
- Медиа объекты должны занимать «разумное» количество пространства на листе
- На каждый медиа объект **должна быть ссылка в тексте**, указанная **до появления объекта**
- Каждый медиа объект **должен быть подписан** — безымянные объекты не допускаются

#### 3.2.1 Оформление рисунков

- Каждый рисунок подписывается
- Подпись располагается **снизу** рисунка
- Рисунок выравнивается по центру
- Подпись рисунка выравнивается по центру
- Каждый рисунок нумеруется с префиксом номера раздела. Пример: `Рис. 3.1`
- Нумерация по порядку в пределах раздела, в новом разделе начинается заново. Пример: `Рис. 3.10`, `Рис. 4.1`

#### 3.2.2 Оформление таблиц

- Каждая таблица подписывается
- Подпись таблицы **сверху**
- Таблица выравнивается по центру
- Подпись таблицы выравнивается по центру
- Каждая таблица нумеруется с префиксом номера раздела. Пример: `Таблица 3.1`
- Нумерация по порядку в пределах раздела, в новом разделе начинается заново
- Подпись таблицы и её тело (хотя бы первые строки) не должны располагаться на разных листах
- Таблица оформляется черно-белая (цветная допускается, если это необходимо по тексту)

**Пример:**

Таблица 3.1 — Пример оформления таблицы с некоторыми данными.

| Год  | Версия Java SE | Конец LTS    |
|------|----------------|--------------|
| 1996 | 1.0            | —            |
| 1997 | 1.1            | —            |
| 2004 | 5.0            | апрель 2015  |
| 2019 | 13             | —            |

#### 3.2.3 Оформление программного кода

- Добавляется подпись листинга **сверху**
- Каждый листинг нумеруется с префиксом номера раздела. Пример: `Листинг 3.1`
- Нумерация по порядку в пределах раздела, в новом разделе начинается заново
- Стиль оформления кода свободный

**Пример:**

Листинг 3.1 — Пример оформления листинга с некоторым кодом.

```java
public class Duck
{
    public int age;
    public string color;
    public string gender;

    public Duck(int age, string color, string gender)
    {
        this.age = age;
        this.color = color;
        this.gender = gender;
        System.out.printf("I born " + GetHashCode());
    }

    public void voice()
    {
        System.out.printf("kra kra age: %d, color: %s, g: %s", age, color, gender);
    }

    public void fly()
    {
        System.out.println("Flyyyyyyy");
    }
}
```

## 4. Правила оформления основных разделов отчета

Отчет имеет следующую структуру:

1. Содержание
2. Введение *(в отчете по практике по желанию)*
3. Основная часть
4. Заключение *(в отчете по практике по желанию)*
5. Список использованных источников
6. Приложения

### 4.1 Оформление содержания

Содержание содержит ссылки на номер страницы каждого раздела, подраздела и т.д.

- Шрифт Times New Roman
- Размер 14
- Межстрочный интервал 1,5
- Выровнен по ширине

Нумерация страниц начинается с титульного листа, но **номер 1 страницы не указывается**. Нумерация располагается справа внизу страницы чёрного или серого цвета.

### 4.2 Оформление введения

Введение позволяет составить общее представление о работе, понять какие задачи стоят перед автором и какие пути их решения он видит.

- Заголовок введения оформляется **без номера**
- Правила оформления как у основного текста
- Объём введения: не более 2 листов, не менее половины листа

### 4.3 Основная часть

Основная часть содержит весь материал работы, включая рисунки, таблицы, программный код и т.д. Заголовки нумеруются по правилам, приведённым ранее.

### 4.4 Заключение

- Заголовок заключения оформляется **без номера**
- Правила оформления как у основного текста
- Объём заключения: не более 2 листов, не менее полулиста

### 4.5 Оформление списка использованных источников

Заголовок списка источников оформляется **без номера**.

- Ссылки на источники указываются в тексте в виде затекстовых ссылок: `[1]`, `[2]`, `[3]`
- Ссылки нумеруются по порядку, начиная с 1
- Не допускается нарушения порядка нумерации
- Повторное использование номера ссылки допускается только после того, как ссылка уже была объявлена в тексте

**Список источников оформляется по ГОСТу. Пример:**

1. Краснова, Л.П. Бухгалтерский учет [Текст]: учебник для вузов / Л.П. Краснова, Н.Т. Шалашова, Н.М. Ярцева. — Москва: Юристъ, 2001. — 550 с.
2. Ашервуд Б. Азбука общения / Б. Ашервуд; пер. с анг. И.Ю. Багровой и Р.З. Пановой, науч. ред. Л.М. Иньковой. — Москва: Либерея, 1995. — 175 с.
3. Исследовано в России [Электронный ресурс]: многопредмет. науч. журн. / Моск. физ.-техн. ин-т. — Электрон. журн. — Долгопрудный: МФТИ, 1998. — режим доступа: http://zhurnul.milt.rissi.ru (дата обращения 25.05.2020)

### 4.6 Оформление приложений

Заголовок приложения указывается с номером (цифры или буквы), после чего может идти описание.

**Примеры заголовков:**
- Приложение А
- Приложение 1
- Приложение Б — пример оформления заголовков

**Приложение, кроме заголовков, оформляется свободно.** На все добавленные в работу приложения должны быть ссылки в тексте (Смотри Приложение А).
